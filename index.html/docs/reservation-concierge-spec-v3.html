概要・改修目的

旧UIの世界観（ミニふじくん案内型・シナリオ進行型）を完全に復元した上で、以下を新たに追加します。

  • コンシェルジュらしい会話トーン・寄り添い案内
  • 症状・年代・性別によるコース/オプション表示ロジック
  • メンズブースト等の限定条件管理
  • 時間表示の改善（分→時間）
  • 将来の追加・変更に強いデータ設計
01

UIデザイン修正方針

1-1. デザインコンセプト（旧UIへの回帰 + ブラッシュアップ）

以下の要素を必ず復元すること（旧UIのコア要素）：

  • 画面上部に「予約コンシェルジュ」タイトル + 「順番に選ぶだけで予約できます」サブテキスト
  • ミニふじくんのキャラクターアイコン（左上または案内エリア左）
  • ミニふじくんの吹き出し型案内テキスト（各ステップで変わる）
  • 黒〜濃いグレー基調のヘッダー/フッターデザイン
  • 大きく押しやすい「はじめる ▶」ボタン
  • 画面下部固定の合計エリア（合計時間・合計金額・現在の選択内容）
  • 「最初からやり直す」リンク（下部に小さく）
  • ステップバイステップで順番に進む設計

ブラッシュアップのみ行う点（変えてよい点）：

  • 選択肢カードを押しやすく（タップ領域拡大）
  • バッジ表示を明確に： NEW 人気No.1 リピNo.1
  • スマホ余白・文字サイズの視認性向上
  • 「戻る」「最初からやり直す」をわかりやすく
変えてはいけない点（旧UIの世界観）： ・黒〜濃いグレー基調の重厚感
・キャラクター案内型UI
・下部固定の進捗・合計表示

1-2. 画面構成（各エリアの仕様）

エリア 仕様
ヘッダーエリア 背景色：#1a1a1a（旧UIに合わせた濃い黒）
タイトル：「予約コンシェルジュ」（中央配置、白文字、18px）
サブテキスト：「順番に選ぶだけで予約できます」（12px、グレー）
ミニふじくんアイコン：左端に32px表示
案内エリア アイコン：ミニふじくん 48px（左）
吹き出し：角丸、背景白、左向き

選択肢エリア カードグリッド：1列（スマホ）/ 2列（PC）
バッジ：右上角に色付きラベル（NEW=青、人気No.1=赤、リピNo.1=緑）
タップ時：背景が変わりチェック表示

03

コース/オプション表示ロジック

3-1. 表示フィルタリング（表示可否の判定）

まず以下の条件で表示するかどうかを判定します：

// 表示条件判定
性別チェック：course.gender が "all" または 選択した性別を含む
年齢上限チェック：course.ageMax が未設定 または 選択した年代の上限以下
年齢下限チェック：course.ageMin が未設定 または 選択した年代の下限以上
重要な限定条件： ・メンズブーストは gender: ["male"]、ageMax: 49
・50代・60代以上には表示しない
・女性限定コースは male には表示しない
・男性限定コースは female には表示しない

3-2. スコアリングと表示順（フィルタ通過後のソート）

// スコア計算式
totalScore = symptomScore + segmentScore + badgeScore + displayPriority
  • symptomScore：選択した症状と symptomWeights の合計
  • segmentScore：性別＋年代のキーに対応する segmentWeights の値
  • badgeScore：NEW=10、人気No.1=8、リピNo.1=6、なし=0
  • displayPriority：手動設定の優先度（0〜100）

最終ソート順：totalScore 降順（高いほど上に表示）

時間表示ルール

分から時間への変換関数仕様：

function formatDuration(minutes) {
  if (minutes < 60) {
    return minutes + "分";
  }
  let hours = Math.floor(minutes / 60);
  let remainder = minutes % 60;
  
  if (remainder === 0) {
    return hours + "時間";
  } else {
    return hours + "時間" + remainder + "分";
  }
}

  • 最終確認サマリー画面の合計時間
06

将来の追加・変更手順

6-1. 新コースを追加する手順

  1. 1
    courses 配列に新しいオブジェクトを追加する
  2. 2
    id（一意のID）、name、duration、price を設定する
  3. 3
    gender（all / male / female）を設定する
  4. 4
    ageMin / ageMax で年齢制限を設定する（制限なしはnull）
  5. 5
    symptomWeights に関連症状とスコアを設定する
  6. 6
    segmentWeights に年代ごとのスコアを設定する
  7. 7
    badges を設定する（NEW / 人気No.1 / リピNo.1 / なし）
  8. 8
    displayPriority を 0〜100 で設定する（高いほど優先）
  9. 9
    isActive を true にする
6-2. 限定条件の設定例： ・49歳以下限定：ageMax: 49
・50代以上限定：ageMin: 50
・男性限定：gender: ["male"]
・女性限定：gender: ["female"]
